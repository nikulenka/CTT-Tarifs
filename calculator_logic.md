# Спецификация бизнес-логики и математики калькулятора

## 1. Каскадная (ступенчатая) тарификация
Критическое правило: расчет стоимости документов работает как прогрессивная шкала. Вы НЕ умножаете весь объем на одну ставку.
**Алгоритм для N документов:**
1. Пройти по массиву `piece_rate_brackets`.
2. Найти вместимость каждого диапазона (например, для 1-100 вместимость 100. Для 101-200 вместимость 100).
3. "Откусывать" от N документы, помещающиеся в диапазон, и умножать их на ставку этого диапазона.
*Пример проверки для агента:* Если на вход дано 223 эТТН, функция должна рассчитать: (100 * 0.39) + (100 * 0.28) + (23 * 0.22) = 96.06. В UI должна выводиться именно эта формула строкой для прозрачности.

## 2. Алгоритм Автоподбора
Если пользователь выбрал "Автоподбор", система выполняет симуляцию:
1. Исключает тарифы, недоступные текущему клиенту (например, "Оптимальный" недоступен действующим).
2. Для каждого доступного тарифа берет абонентскую плату.
3. Вычитает из введенного объема эТТН/эТН включенный лимит тарифа (если есть).
4. На оставшийся объем применяет функцию каскадной тарификации (или перебирает покупку пакетов из JSON, находя наименьшую сумму для остатка).
5. Сравнивает `Итог = Абонентская плата + Оплата сверх лимитов`.
6. Рекомендует тариф с минимальным Итогом.

## 3. Трансграничные пакеты
Поштучной тарификации для ТEDI и ТЭДО нет. Алгоритм должен подобрать минимальную комбинацию пакетов из массива `cross_border`, чтобы покрыть введенный пользователем объем (например, для 120 документов выгоднее взять 1 пакет на 100 и 1 на 50, или 1 на 500 — скрипт должен сравнить их цены).
