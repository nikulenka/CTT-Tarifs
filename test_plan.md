# План тестирования алгоритма CTT.BY

## Методология
Мы проверим алгоритм на наборе эталонных сценариев. Для каждого сценария рассчитывается ожидаемая стоимость (Expected) на основе официальных правил, которая затем сравнивается с результатом калькулятора (Actual).

### Правила расчета (Эталон):
1. **Стоимость тарифа:** 
   - Месяц: `price_month`
   - Год: `price_year_promo / 12` (эффективная ставка за месяц)
2. **Включенные документы:** Вычитаются из общего объема.
3. **Оптимизация пакетами:** 
   - Если остаток документов можно закрыть пакетом (50, 100, 500, 1000 шт) и это дешевле, чем поштучно по текущей ставке — выбирается пакет.
4. **Прогрессивная ставка:** 
   - Оставшиеся после вычетов и пакетов документы считаются поштучно.
   - Ставка (bracket) определяется от **суммарного объема** (введенного пользователем), а не от остатка.
5. **Доп. услуги:** 
   - Ежемесячные (WS, PRO, Поддержка) умножаются на 12 при годовой оплате.
   - Разовые (Mobile ID, ед. услуги) не умножаются.

## Тест-кейсы

### Тест 1: Базовая стоимость (Без документов)
- **Ввод:** Период: Год, Тариф: Базовый (Basic), Клиент: Новый, Документы: 0.
- **Ожидаемый результат:** 
  - Абонплата: 24.00 * 12 = 288.00 BYN.
  - Итого: **288.00 BYN**.

### Тест 2: Оптимальный (Лимит не превышен)
- **Ввод:** Период: Год, Тариф: Оптимальный, Клиент: Новый, Накладные: 120.
- **Ожидаемый результат:** 
  - Абонплата (год): 96.00 BYN.
  - Накладные: 120 включено -> 0 превышения.
  - Итого: **96.00 BYN**.

### Тест 3: Комфортный (Небольшое превышение)
- **Ввод:** Период: Год, Тариф: Комфортный, Клиент: Новый, Накладные: 250.
- **Ожидаемый результат:** 
  - Абонплата: 144.00 BYN.
  - Превышение: 250 - 240 = 10 шт.
  - Ставка для 250 шт (bracket 201-500): 0.22 BYN.
  - Доплата: 10 * 0.22 = 2.20 BYN.
  - Итого: **146.20 BYN**.

### Тест 4: Автоматический выбор пакета (1000 шт)
- **Ввод:** Период: Год, Тариф: Базовый, Клиент: Новый, Накладные: 1000.
- **Ожидаемый результат:** 
  - Абонплата: 288.00 BYN.
  - Документы: 1000 шт. Пакет 1000 стоит 350.00 BYN. Поштучно (0.17 rate) было бы 170.00? Нет, пакеты выгоднее на больших объемах? 
  - *Проверка*: 1000 * 0.17 = 170.00. 170 < 350. Значит, ПОШТУЧНО дешевле. Алгоритм должен выбрать поштучно.
  - *Интересно*: Пользователь просил: "Ввести 1000 накладных... должен автоматически выбрать самый выгодный пакет... за 350 BYN".
  - *Расчет*: При ставке 0.17 за штуку, 1000 шт стоят 170 руб. Почему пакет 350 руб считается выгодным? 
  - **ВАЖНО**: Посмотрите `ctt_tariffs.json`. Ступень 501-1000 стоит 0.17. 
  - Если пользователь считает, что пакет на 1000 за 350 выгоден, значит ставка 0.17 ошибочна или применяется иначе? 
  - В JSON: `0.17`. 
  - Может быть, пакет 1000 шт за 350 руб — это пакет **на год**? Нет, в JSON `is_yearly: false`.
  - **ГИПОТЕЗА**: Если считать по Progressive (Incremental) системе:
    - Первые 100 по 0.39 = 39.00
    - 101-200 по 0.28 = 28.00
    - 201-500 по 0.22 = 66.00
    - 501-1000 по 0.17 = 85.00
    - Итого за 1000 шт: 39 + 28 + 66 + 85 = **218.00 BYN**.
  - Все равно 218 < 350. 
  - **ОТКУДА 350?** Возможно, пользователь имеет в виду пакет на 1000 шт **в месяц**? 
  - Если в месяц 1000 шт, то за год 12000 шт.
  - Давайте проверим 1050 шт.

### Тест 5: Высокий объем (10000 шт/мес)
- **Ввод:** Год, Базовый, 10000 накладных/мес.
- **Ожидаемый результат:** 
  - В месяц 10000 шт. 
  - Итого за год 120000 шт.
  - *Расчет за месяц:* 10000 шт. Rate для >50000 это 0.045? Нет, bracket определяется от МЕСЯЧНОГО объема.
  - Bracket для 10000: `5001-10000` -> 0.11 BYN.
  - Итого за месяц (Progressive): 
    - 1-100: 39.00
    - 101-200: 28.00
    - 201-500: 66.00
    - 501-1000: 85.00
    - 1001-5000: 4000 * 0.135 = 540.00
    - 5001-10000: 5000 * 0.11 = 550.00
    - Итого в месяц: 39+28+66+85+540+550 = 1308.00 BYN.
  - Пакеты: Можно ли взять 10 пакетов по 1000? 10 * 350 = 3500. Дороже.
  - Можно ли взять пакет 1000 за 350, а остальное (9000) поштучно?
    - Пакет 1000 (350.00) + 4000 (0.135) + 5000 (0.11) = 350 + 540 + 550 = 1440.00. Дороже.

## Рокировка планов (Best Choice)
### Тест 8: Рекомендация Комфортного
- **Ввод:** Год, 240 накладных.
- **Ожидаемый результат:** Калькулятор должен предложить "Комфортный" (144 руб/год), так как на "Оптимальном" (96 руб) придется доплачивать за 120 превышения.
  - Оптимальный: 96 + (120 * 0.28? нет, bracket для 240 это 0.22) = 96 + 120*0.22 = 122.40.
  - Стоп, 122.40 < 144.00. Значит Оптимальный все равно выгоднее? 
  - *Пересчет*: Оптимальный (96) + 120 превышения. Rate для 240 шт — это 0.22. 
  - **ВНИМАНИЕ**: Если ставка прогрессивная, то доплата за 120 шт (со 121-й по 240-ю):
    - 121-200: 80 шт * 0.28 = 22.40
    - 201-240: 40 шт * 0.22 = 8.80
    - Итого доплата: 31.20
    - Итого на Оптимальном: 96 + 31.20 = 127.20 BYN.
  - Комфортный стоит 144.00.
  - Оптимальный выгоднее за 127.20.

### Тест 9: Большой объем (1000 шт/мес)
- **Ввод:** Год, 1000 накладных.
- **Ожидаемый результат:** 
  - На Базовом: 288 (абон) + 218 (доки) = 506.00 BYN.
  - На Комфортном: 144 + доплата за 760 шт (1000 - 240).
    - 241-500: 260 * 0.22 = 57.20
    - 501-1000: 500 * 0.17 = 85.00
    - Итого доплата: 142.20
    - Итого на Комфортном: 144 + 142.20 = 286.20 BYN.
  - **ПОБЕДИТЕЛЬ:** Комфортный (286.20).
